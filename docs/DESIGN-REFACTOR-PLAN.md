# 设计系统重构计划

参考 Notion 的视觉体系：黑白灰为主，彩色为低饱和度点缀，整体统一不杂乱。

## 执行前提

- 在 worktree 分支做，不在 master 上改
- 每步完成后截图对比，确保没有意外变化

## 第一步：CSS 变量化（不改视觉）

1. 在 `main.css` 定义 CSS 变量（暗色值和现在完全一样）
2. 逐个组件替换硬编码 className 为变量引用
3. 完成后截图，必须和改之前一模一样
4. commit："CSS 变量化：不改视觉，为主题切换做准备"

变量命名参考：
```css
:root {
  /* 暗色主题（默认） */
  --bg-base: #18181b;        /* zinc-900 */
  --bg-surface: #27272a;     /* zinc-800 */
  --bg-hover: #3f3f46;       /* zinc-700 */
  --text-primary: #e4e4e7;   /* zinc-200 */
  --text-secondary: #a1a1aa; /* zinc-400 */
  --text-muted: #71717a;     /* zinc-500 */
  --text-faint: #52525b;     /* zinc-600 */
  --border-default: #3f3f46; /* zinc-700 */
  --border-subtle: #27272a;  /* zinc-800 */
  --accent: #3b82f6;         /* blue-500，选中态 */
  --active: #4ade80;         /* green-400，活跃状态 */
}
```

## 第二步：统一色板（视觉会变）

参考 Notion 的低饱和色板，把"五彩斑斓"收敛为：
- 灰度基底（所有文字、背景、边框）
- 语义色只保留：蓝（选中/链接）、绿（活跃/成功）、红（删除/错误）、黄（警告/搜索高亮）
- 标签/badge 用 Notion 风格的低饱和色（柔和，不刺眼）
- 工具调用颜色：统一改为灰色底 + 工具名文字区分（或保留彩色但降低饱和度，由用户决定）

需要用户确认的决策：
- [ ] 工具调用标签保留彩色还是改灰色？
- [ ] 用户头像/助手头像用颜色区分还是用图标区分？
- [ ] compact/分支标签保留独立颜色还是统一为灰色？

## 第三步：加浅色模式

1. 在 CSS 变量里加一组浅色值
2. 加切换按钮（放在 Toolbar 里，已有语言切换，加一个主题切换）
3. 用 `prefers-color-scheme` 做默认值，手动切换后记住选择
4. E2E 截图覆盖两种模式

## 风险控制

- 每步结束 commit，可独立回滚
- 第一步截图 = 基准线，后续步骤偏离时有参照
- 不确定的设计决策先问用户，不要自己判断
